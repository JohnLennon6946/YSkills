# crowd-async-fetch

异步拉取特定任务类型的今日人群包，含提交任务和轮询结果的完整流程。

## 前置条件

1. mws 已新增人群拉取和任务结果查询两个接口（P2 阶段）
2. mws CLI 已安装且已认证

## 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 计划类型：签到/高耗币/礼物过期/家族签到/私聊 |

## 执行流程

> **P0/P1 阶段**：人群拉取接口尚未上线，直接返回：
> "人群拉取接口暂未上线，请通过模式 A 手动提供 crowdPacketUrl 创建计划。"

> **P2 阶段实现**（以下为预留流程）：

### 步骤 1：提交拉取任务

```bash
# 接口待 mws 新增
mws <service> <submit-method> --json '{"type": "{type}"}'
```

从响应中获取 `taskId`。

### 步骤 2：轮询任务结果

每 10 秒调用一次结果查询接口：

```bash
# 接口待 mws 新增
mws <service> <result-method> --json '{"taskId": "{taskId}"}'
```

轮询逻辑：
- 任务状态为"执行中"：等待 10 秒后再次查询
- 任务状态为"完成"：从响应中提取 `crowdPacketUrl`，返回成功
- 任务状态为"失败"：停止轮询，返回失败原因
- 累计轮询超过 5 分钟（30 次）：停止轮询，返回超时错误

### 步骤 3：返回结果

- 成功：返回 `crowdPacketUrl` 字符串
- 失败：返回结构化错误信息（含失败原因或超时说明）

## 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 接口未上线（P0/P1） | 返回不可用提示 |
| 任务提交失败 | 返回接口错误信息 |
| 任务超时（5 分钟） | 停止轮询，返回超时错误 |
| 任务返回失败状态 | 停止轮询，返回失败原因 |
| 网络错误 | 单次轮询失败不中断，继续下一次轮询 |
