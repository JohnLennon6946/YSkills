# wechat-push

微信 Push 计划管理主入口 Skill。统一接收用户消息和定时任务触发，完成意图识别和权限校验后，分发到对应的执行 Skill 或直接处理配置管理指令。

## 前置条件

1. mws CLI 已安装且已认证（`mws auth status` 正常）
2. 有 moyi-activity-backend 的 secretary-page 和 secretary-create 接口权限
3. `wechat-push-create/config.json` 位于同级 Skill 目录下

## 意图识别

收到消息后，按以下优先级判断意图并分发：

| 优先级 | 意图 | 判断条件 | 处理方式 |
|--------|------|----------|----------|
| 1 | 配置管理 | 消息包含"修改排期"、"新增场景"、"删除场景"、"查看排期"、"设置业务管理员"、"添加业务管理员"、"移除业务管理员"、"查看业务管理员" | 本 Skill 直接处理 |
| 2 | 模式 A（创建计划） | 消息包含 nosKey（形如 `jdmosi-common/obj/...`）或 txt 文件附件，且提到预设类型之一 | 校验权限后分发到 wechat-push-create |
| 3 | 模式 C（通用创建） | 消息要求手动指定参数或创建非预设类型 | 校验权限后分发到 wechat-push-create |
| 4 | 模式 D（半自动任务） | 消息包含"执行今天的微信push任务"等类似表达，或内部调用传入 `mode: "D"` | 类似模式B的全自动主流程，但增加人群包确认步骤，让用户double-check后再继续 |
| 5 | 模式 B（定时任务） | 由 cron 定时触发，或内部调用传入 `mode: "B"` | 分发到 wechat-push-schedule |

模式 A 中类型不明确时，主动追问用户："请问要创建哪种类型的 push 计划？（签到/高耗币/礼物过期/家族签到/私聊）"

## 计划类型（可动态新增）

| 类型 | planName 关键词 | 说明 |
|------|----------------|------|
| 签到 | `签到` | 每日签到提醒 |
| 高耗币 | `高耗币` | 高耗币用户召回 |
| 礼物过期 | `礼物过期` | 礼物过期提醒 |
| 家族签到 | `家族签到` | 家族签到提醒 |
| 私聊 | `私聊` | 私聊推送 |

类型可通过「新增场景」指令动态扩展，新增类型自动纳入定时调度。

## 权限模型（两级管理员）

**系统管理员**：
- 来源：POPO 机器人后台的子管理员列表（平台自动同步，不存储在 config.json）
- 权限：全部操作（配置业务管理员、管理排期、新增/删除场景、创建 push 计划）

**业务管理员**：
- 来源：由系统管理员通过对话指令配置，存储在 config.json 的 `businessAdmins` 字段
- 权限：管理排期（修改排期、新增/删除场景）、创建 push 计划

**普通用户**：
- 权限：查看排期、查看业务管理员、创建 push 计划（模式 A/C）

**权限校验流程**：
1. 从 POPO 消息获取发送者身份（邮箱）
2. 查询 POPO 机器人后台子管理员列表 → 若命中，视为系统管理员，放行所有操作
3. 若非系统管理员，查找 config.json 的 `businessAdmins` → 若命中，放行业务管理操作
4. 均未命中 → 对需要权限的操作拒绝：
   ```
   抱歉，您没有该操作的权限。请联系系统管理员分配权限。
   ```

查看类指令（查看排期、查看业务管理员）和创建 push 计划（模式 A/C）不限制权限，任何人均可执行。

## 配置管理（本 Skill 直接处理）

配置文件路径：`YSkills/skill/wechat-push-create/config.json`

### 指令处理

| 指令 | 操作 | 权限要求 |
|------|------|----------|
| `修改排期 高耗币 周一周三` | 更新指定类型的执行日 | 系统管理员 或 业务管理员 |
| `新增场景 拉新` | 追问执行周期，写入 schedule 新类型 | 系统管理员 或 业务管理员 |
| `删除场景 拉新` | 从 schedule 中移除该类型 | 系统管理员 或 业务管理员 |
| `查看排期` | 读取并展示排期表（类型 + 对应星期） | 所有人 |
| `设置业务管理员 user1 user2` | 覆盖写入 businessAdmins | 系统管理员 |
| `添加业务管理员 user3` | 追加到 businessAdmins（去重） | 系统管理员 |
| `移除业务管理员 user1` | 从 businessAdmins 删除 | 系统管理员 |
| `查看业务管理员` | 展示当前业务管理员列表 | 所有人 |

星期解析规则：周日=0, 周一=1, 周二=2, 周三=3, 周四=4, 周五=5, 周六=6。支持"每天"作为 [0,1,2,3,4,5,6] 的快捷方式。

### 新增场景交互流程

当系统管理员或业务管理员说"新增场景 {类型名}" 时：

1. 从指令中提取类型名称（如"拉新"）
2. 追问执行周期：`请问「拉新」类型的执行周期是？（如：每天 / 周一周三 / 周二周四周六）`
3. 解析星期数组，写入 config.json 的 schedule 字段
4. 确认并展示完整排期表

### 查看排期展示格式

```
当前排期表：
- 签到：每天
- 高耗币：周一
- 礼物过期：周三、周五
- 家族签到：周四、周六
- 私聊：周二、周日
```

## 分发到子 Skill

### 分发到 wechat-push-create

适用于模式 A 和模式 C，传入参数：

| 参数 | 说明 |
|------|------|
| mode | `"A"` 或 `"C"` |
| type | 计划类型（模式 A） |
| crowdPacketUrl / nosKey | 人群包地址（模式 A） |
| 全部手动参数 | 模式 C 收集完成后传入 |

### 分发到 wechat-push-schedule

适用于模式 B（定时任务）和模式 D（半自动任务）。

**模式 B**：由 cron 定时触发，全自动执行，无需额外参数，wechat-push-schedule 自行读取排期配置。

**模式 D**：用户手动触发（如"执行今天的微信push任务"），传入 `mode: "D"` 和当天匹配的 `todayTypes` 列表。类似模式 B 的全自动主流程，但增加人群包确认步骤，让用户 double-check 后再继续。

## 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 意图无法识别 | 提示用户支持的操作类型 |
| config.json 损坏或缺失 | 使用内置默认排期，businessAdmins 为空 |
| 权限不足 | 提示用户联系系统管理员 |
| 当天 20:00 已过（创建计划时） | 提示用户确认是否仍创建 |
