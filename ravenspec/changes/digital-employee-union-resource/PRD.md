# 数字员工-工会自动化资源位分配 产品需求文档

> 创建日期: 2026-06-29

## 需求概述

为多人房资源位推荐编写自动化 Skill（union-resource-recommend），让运营通过自然语言对话触发离线取数、查看推荐结果、并一键下发资源位分配计划到后端。

## 核心功能

### 功能 1：离线取数 + 表格展示（第一步）

**描述**：识别用户消息中的资源位相关关键词，自动提交 unionCoins 类型异步取数任务，轮询等待完成，将约 100 条推荐数据以 Markdown 表格全量展示在对话中。

**输入**：用户在 POPO 群聊/私聊中发送的自然语言消息（含触发词）

**输出**：下载 dataUrl 的 CSV、解析清洗（空值→`-`、金币去 `.0000`）后以 Markdown 表格全量展示；末尾附下载链接。格式见 SKILL.md。

**触发词分层**：

| 优先级 | 类别 | 关键词 |
|--------|------|--------|
| 1 | 明确指令 | 多人房资源位、资源位推荐、资源位分配 |
| 2 | 带域名词 | 工会资源位、union资源位、公会资源位、数字员工资源位 |
| 3 | 模糊查询 | 拉取资源位、查询资源位、获取资源位 |

仅出现"资源位"无上下文 → 追问确认后触发。

**边界条件**：
- 正常情况：用户发含触发词的消息 → 提交任务 → 轮询 → 表格展示
- 异常情况：任务提交失败返回错误；轮询超时（3小时）提示重试；返回数据为空提示无新数据

### 功能 2：资源位分配下发（第二步）

**描述**：第一步表格展示后，用户选择 tab 类型并输入分配指令，逐号位调用 rcmdback-add 创建推荐计划，最后调用 rcmdback-list 汇总展示创建结果。

**Tab 类型**（用户自行输入，接口支持的枚举如下，未提供则追问）：

| 枚举名 | key | 显示名 |
|--------|-----|--------|
| THEME | `theme` | 主题 |
| FAMILY_PLAY | `family` | 家族在玩 |
| FAMILY_VOICE | `family_voice` | 家族语音房 |
| RCMD | `rcmd` | 推荐 |
| CITY | `city` | 同城列表 |
| DAILY_TASK | `task` | 日常任务 |

一次分配中所有号位共用同一个 tab。

**输入**：用户发送的分配指令（含 tab 类型），支持两种房间指定方式：

| 格式 | 示例 | 说明 |
|------|------|------|
| 序号范围 | `tab=主题 1-10号放1号位，时间是2025-01-01~2025-01-07` | 从第一步表格按序号映射房间号 (room_no) |
| 心遇号直指 | `tab=theme 心遇号 12345,12346 放2号位，时间是2025-01-08~2025-01-14` | 直接使用房间号 (room_no)，逗号分隔 |

两种格式可在同一条消息中混合使用。

**输出**：
- 逐条创建进度反馈（含房间数量）
- 最终汇总表格（号位、房间号 (room_no)、tab 类型、生效时间、计划 ID、状态）

**边界条件**：
- 正常情况：选择 tab → 解析指令 → 逐条创建 → 汇总展示
- 异常情况：tab 未提供追问；tab 不在列表提示；序号超范围跳过；创建单条失败继续下一条；全部失败提示检查接口

## 用户场景

### 场景 1：运营拉取今日资源位推荐

**入口**：POPO 群聊 @机器人 或 私聊机器人
**触发条件**：用户发送含触发词的消息，如"帮我拉一下今天的多人房资源位"
**操作流程**：
1. 用户发送触发消息
2. 机器人提交 unionCoins 异步取数任务，返回进度提示
3. 轮询至任务完成（通常几分钟内）
4. 机器人将 ~100 条推荐数据以 Markdown 表格输出到对话中
5. 用户查看表格，准备下发

### 场景 2：运营下发资源位分配

**入口**：第一步表格展示后的同一条对话线程
**触发条件**：用户发送分配指令
**操作流程**：
1. 用户编写分配指令："1-10号放1号位，时间是2025-06-29~2025-07-05 心遇号 88888,99999 放2号位，时间是2025-07-06~2025-07-12"
2. 机器人解析指令，输出逐条创建进度
3. 全部创建完成后，汇总表格展示

## 边界条件

| 场景 | 处理方式 |
|------|----------|
| mws 未安装或未认证 | 返回「mws 未认证，请先执行 mws auth」 |
| open-task-process 接口失败 | 返回原始错误信息，不重试 |
| processId 为空 | 返回「任务提交失败，未获取到有效的任务 ID」 |
| 轮询超时（3 小时） | 返回超时提示，附带 processId 供排查 |
| dataUrl 返回但无法访问 | 返回「数据拉取完成但下载链接不可用」，附带 dataUrl 和 processId |
| 返回数据为空 | 返回「本次未拉取到资源位推荐数据」 |
| 返回字段与预期不一致 | 以实际字段为准动态生成表头，提示用户核对 |
| 用户仅说"资源位"无上下文 | 追问确认是否拉取多人房资源位数据 |
| tab 类型未提供 | 追问展示 6 个可选 tab，等待用户选择或自行输入 |
| tab 类型不在已知枚举 | 允许用户自行输入，透传给接口 |
| 用户分配指令无法解析 | 追问并给出格式 A 和格式 B 的示例 |
| 序号超出第一步数据范围 | 提示超出（最多 N 条），跳过该条继续 |
| 号位重复 | 提示用户该号位已分配，询问是否覆盖 |
| rcmdback-add 某条创建失败 | 记录失败原因，继续下一条 |
| rcmdback-add 全部创建失败 | 返回「所有号位创建均失败，请检查接口状态」 |
| rcmdback-list 查询失败 | 从 add 返回值拼接汇总，提示查询接口不可用 |
| open-task-process-status 单次网络错误 | 不中断，继续下一次轮询 |

## 非功能需求

- **轮询策略**：每 1 分钟轮询一次，超时阈值 3 小时（180 次）
- **数据量级**：约 100 条推荐数据，全量单次展示不截断
- **接口依赖**：mws moyi-activity-backend（open-task-process、open-task-process-status、rcmdback-add、rcmdback-list）
- **触发方式**：POPO 消息中关键词匹配触发，非定时任务

## 验收标准

| 功能 | 验收标准 |
|------|---------|
| 触发词识别 | 用户发送"多人房资源位推荐"，机器人开始取数流程 |
| 模糊追问 | 用户仅说"资源位"，机器人追问确认后再触发 |
| 异步取数 | open-task-process 提交后成功返回 processId |
| 轮询完成 | status=done 后 dataUrl 可访问，数据解析为表格 |
| 表格展示 | ~100 条数据以完整 Markdown 表格输出，无截断 |
| tab 选择 | 用户在指令中包含 `tab=唠嗑`，或 Skill 追问后选择 tab |
| 序号分配 | 用户说"1-10号放1号位"，查第一步序号映射房间号 (room_no)，创建成功 |
| 心遇号分配 | 用户说"心遇号 12345,12346 放2号位"，直接用 ID 创建，创建成功 |
| 混合分配 | 同一条消息中混合序号和心遇号，全部解析并创建无误 |
| 汇总展示 | rcmdback-list 查询结果以表格输出 |

## 非目标

- 本期不支持定时任务自动触发（需用户手动发起）
- 本期不支持对推荐结果做编辑/调整后下发
- 本期仅支持多人房资源位，不支持 banner 等其他类型资源位
- 本期不支持资源位分配的撤销/修改（创建后需后台操作）

## Capabilities

### 新增能力

- `union-resource-fetch`: 触发词识别 + 离线取数（open-task-process → 轮询 open-task-process-status） + 结果解析与 Markdown 表格展示
- `union-resource-dispatch`: tab 类型选择 + 用户分配指令解析（序号范围 / 心遇号直指） + 逐号位 rcmdback-add 创建 + rcmdback-list 汇总展示

### 修改能力

_无_
