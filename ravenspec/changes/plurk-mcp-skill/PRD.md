# plurk-mcp-skill 产品需求文档

> 创建日期: 2026-03-20

## 需求概述

为 OpenClaw AI Agent 提供一套 Plurk 平台操作能力的 MCP Server 及配套 Skill，使 AI Agent 能够每天定时自动登录 Plurk、发布帖子、并基于 AI 生成的文案自动回复他人的帖子。

## 核心功能

### 功能 1：Plurk 账号认证

**描述**：支持两种认证方式让 MCP Server 获得 Plurk 平台的操作权限：
1. **OAuth 1.0a 模式**：通过 Plurk 官方 API 的 OAuth 认证获取 access token，适合长期稳定运行。
2. **账号密码模拟登录模式**：通过 requests session 模拟浏览器登录，适合不便申请 OAuth App 的场景。

认证凭据（API Key / Access Token / 用户名密码）通过环境变量或 `.env` 文件注入，不硬编码。

**边界条件**：
- 正常情况：认证成功后，将 session/token 缓存到本地，后续调用复用，无需每次重新登录。
- 异常情况（token 过期/session 失效）：自动触发重新认证，重试当前操作；若重试失败则返回明确错误信息给调用方。
- 异常情况（凭据错误）：立即返回认证失败错误，不重试，提示用户检查配置。

---

### 功能 2：发布新帖子（Plurk）

**描述**：AI Agent 调用此工具，向 Plurk 发布一条新帖子。

- **输入**：帖子正文（纯文本或含 emoji）、qualifier（如 `:` / `says` / `shares` 等，默认 `:`）
- **输出**：发帖成功后返回帖子的 `plurk_id` 和帖子 URL
- **定时发帖**：本 MCP 不内置定时器，定时调度由 OpenClaw Agent 侧负责（如 cron / scheduler）；MCP 只负责执行单次发帖动作。

**边界条件**：
- 正常情况：调用 Plurk API 发帖成功，返回 plurk_id。
- 异常情况（内容为空）：返回参数校验错误，拒绝调用。
- 异常情况（内容超过 360 字符）：返回超长错误，提示调用方截断或分段。
- 异常情况（API 限流）：返回限流错误及建议重试时间（若 API 有提供）。
- 异常情况（网络错误）：重试最多 3 次，每次间隔 5 秒，仍失败则返回错误。

---

### 功能 3：回复帖子

**描述**：AI Agent 调用此工具，对指定 `plurk_id` 的帖子下、针对特定 `user_id` 发布一条 @ 回复。回复文案由 AI Agent 在调用前自行生成，作为参数传入。

- **输入**：
  - `plurk_id`：目标帖子的 ID
  - `user_id`：被回复的目标用户 ID（用于在回复内容中自动拼接 `@username`）
  - `content`：回复正文（不含 @ 前缀，由 MCP 自动根据 `user_id` 查询用户名并拼接）
- **输出**：回复成功后返回 `response_id`
- **说明**：MCP 需先通过 `user_id` 查询对应的 `display_name` / `nick_name`，再将 `@username content` 作为最终回复内容发送到 Plurk API。

**边界条件**：
- 正常情况：回复发送成功，返回 response_id。
- 异常情况（plurk_id 不存在或已被删除）：返回明确错误，不重试。
- 异常情况（user_id 无效或查询不到对应用户名）：返回用户不存在错误，拒绝发送。
- 异常情况（无权限回复，如对方设置了限制）：返回权限错误。
- 异常情况（content 为空）：返回参数校验错误。

---

### 功能 4：获取时间线（Timeline）

**描述**：获取当前账号的 Plurk 时间线，用于 AI Agent 了解近期动态，决定是否需要回复。

- **输入**：可选的分页偏移（offset，默认获取最新 20 条）
- **输出**：帖子列表，每条包含 `plurk_id`、发帖人、内容、发帖时间、回复数

**边界条件**：
- 正常情况：返回最新帖子列表。
- 异常情况（无数据）：返回空列表，不报错。

---

### 功能 5：获取帖子回复列表

**描述**：获取指定帖子下的所有回复，用于 AI Agent 了解上下文后生成更合适的回复文案。

- **输入**：目标帖子的 `plurk_id`
- **输出**：回复列表，每条包含 `response_id`、发帖人、内容、时间

**边界条件**：
- 正常情况：返回回复列表（可能为空列表）。
- 异常情况（plurk_id 不存在）：返回错误。

---

### 功能 6：删除自己的帖子

**描述**：删除当前账号发布的指定帖子，用于 Agent 纠错或内容管理。

- **输入**：目标帖子的 `plurk_id`
- **输出**：删除成功/失败状态

**边界条件**：
- 异常情况（非本人帖子）：返回权限错误，不执行删除。
- 异常情况（帖子不存在）：返回不存在错误。

---

### 功能 7：获取用户资料

**描述**：获取指定用户（或当前登录账号）的基本资料，供 AI Agent 了解互动对象。

- **输入**：用户名（可选，不传则返回当前登录账号资料）
- **输出**：用户昵称、头像 URL、关注数、粉丝数、个人简介

---

### 功能 8：点赞帖子

**描述**：对指定帖子执行点赞（Like）操作。

- **输入**：目标帖子的 `plurk_id`
- **输出**：操作成功/失败状态

---

### 功能 9：MCP Skill 文档

**描述**：提供一个 `SKILL.md` 文件，指导 AI Agent（如 OpenClaw）如何配置和使用本 MCP Server，包括：
- 环境变量配置说明
- OAuth vs 模拟登录的选择指引
- 常用场景的调用示例（定时发帖、自动回复）
- 错误处理建议

---

## 用户场景

### 场景 1：每日定时自动发帖

**入口**：OpenClaw 的 cron/scheduler 触发 AI Agent
**触发条件**：每天指定时间（如早上 9:00）
**操作流程**：
1. OpenClaw scheduler 触发 Agent 执行「每日发帖」任务
2. Agent 调用内容生成逻辑，产出当天的帖子文案
3. Agent 调用 MCP Tool `plurk_post`，传入文案内容
4. MCP 执行发帖，返回 `plurk_id` 和帖子 URL
5. Agent 记录发帖结果，任务结束

---

### 场景 2：AI 自动回复时间线上的帖子

**入口**：OpenClaw Agent 定时任务或手动触发
**触发条件**：Agent 决定对某条帖子进行互动
**操作流程**：
1. Agent 调用 MCP Tool `plurk_get_timeline`，获取最新帖子列表
2. Agent 选取需要回复的帖子（根据内容相关性、优先级等策略）
3. Agent 调用 MCP Tool `plurk_get_responses`，获取该帖子的现有回复，了解上下文
4. Agent 基于帖子内容和上下文，自行生成回复文案
5. Agent 调用 MCP Tool `plurk_reply`，传入 `plurk_id` 和生成的回复文案
6. MCP 发送回复，返回 `response_id`

---

### 场景 3：初始化认证配置

**入口**：用户首次部署 MCP Server
**触发条件**：首次运行或凭据变更
**操作流程**：
1. 用户参考 `SKILL.md` 说明，在 `.env` 文件中填写认证信息
2. 用户选择认证模式（OAuth 或账号密码）
3. MCP Server 启动时自动读取配置并完成认证
4. 认证成功后缓存 session/token，后续调用无需重复认证

---

## 边界条件

| 场景 | 处理方式 |
|------|----------|
| Plurk API 限流（429） | 返回限流错误及 retry-after 建议，由调用方决定重试时机 |
| 网络超时 | 自动重试 3 次（间隔 5s），仍失败返回超时错误 |
| session/token 过期 | 自动触发重新认证，认证成功后重试原操作（最多 1 次） |
| 凭据配置缺失（.env 未配置） | Server 启动时立即报错并打印配置说明，拒绝启动 |
| 发帖内容超过 360 字符 | 参数校验阶段拒绝，返回明确错误信息，不调用 API |
| 操作非本人资源（删除他人帖子） | 返回权限错误，不执行操作 |

---

## 非功能需求

- **性能**：单次 API 调用（发帖/回复）响应时间 < 5s（不含网络往返波动）
- **可靠性**：网络错误自动重试 3 次；关键操作（发帖/回复）失败时返回结构化错误，便于 Agent 侧处理
- **安全性**：认证凭据仅通过环境变量传入，禁止硬编码到代码或提交到 Git
- **兼容性**：Python 3.10+；支持 MCP 标准协议，兼容 OpenClaw 及其他支持 MCP 的 Agent 框架

---

## 非目标（本期不做）

- 不内置定时调度器（cron），定时逻辑由 OpenClaw 侧负责
- 不做 Plurk 私信（Direct Message）功能
- 不做好友关系管理（关注/取关）
- 不做媒体上传（图片/视频附件发帖）
- 不做 Plurk 通知中心的读取
- 本期 Skill 文档仅支持 OpenClaw 接入示例，不覆盖其他 Agent 框架

---

## Capabilities

### 新增能力

- `plurk-auth`: Plurk 账号认证能力，支持 OAuth 1.0a 和账号密码模拟登录双模式，含 session/token 缓存管理
- `plurk-post`: 发布新帖子到 Plurk 平台的能力，含参数校验与错误处理
- `plurk-reply`: 回复指定 Plurk 帖子的能力
- `plurk-timeline`: 获取当前账号时间线帖子列表的能力
- `plurk-responses`: 获取指定帖子回复列表的能力
- `plurk-delete`: 删除当前账号帖子的能力
- `plurk-profile`: 获取用户资料的能力
- `plurk-like`: 点赞帖子的能力
- `plurk-mcp-server`: MCP Server 主入口，将上述能力注册为 MCP Tools 并对外暴露
- `plurk-skill-doc`: 面向 OpenClaw AI Agent 的 Skill 接入文档与使用说明

### 修改能力

_无_
